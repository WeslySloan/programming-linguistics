2. Algol 언어의 주요 특징을 나열하여라.
- 산과정묘사 지향적인 언어이자 알고리즘 묘사 지향적인 언어
- 알고리즘을 표현하기 위한 언어 
- 특정 하드웨어를 기반으로 설계되지 않음
- 언어 문법을 Context-Free Grammar로 표현 (BNF, Backus Naur Form)
- Block Structure (즉, 복합문) 및 변수의 영역(scope) 도입
- 재귀함수
- Call-by-name, Call-by-value 두 가지 파라메터 전송 방식 도입
- Stack 기반의 연산법 제시
- 현대 명령형 언어의 이론적 모델을 제시함

4. 컴파일러의 Scanner(어휘분석기)와 Parser(구문분석기)의 기능을 간략히 설명하라.
- scanner : 코드 내의 token이 어디에 속하는지 분류해주는 역할
- parser : 원시 프로그램의 명령문이나 온라인 명령문, HTML 문서 등에서 Markup Tag 등을 입력으로 받아들여서 구분을 해석 할 수 있는 단위로 여러 부분으로 해석해 주는 역할

6. 다음 물음에 답하여라.

 ( a ) 순수 함수 ( pure functions ) 에 대해 설명하라. ( 참조 투명성과 함께 )
- 참조 상 투명한 함수를 평가하게 되면 동일한 인자에 대해 동일한 값을 반환해야 한다. 그러한 함수를 순수 함수라고 부른다.

 ( b ) Side-effect  함수란 무엇인가? 어떤 경우에 Side-effect가 발생하는가?
- Side-effect : 반환 값 이외에, 호출 된 함수 밖에서 관찰할 수 있는 어플리케이션의 상태 변경
예시로 y = random() * random(); 은 호출할 때마다 결과가 달라질 수 있는 함수이므로 이러한 경우에 발생한다.

 ( c ) Side-effect 를 사용하지 않고 프로그래밍하기 어려운 대표적인 분야는 무엇인가?
- 입출력 및 이벤트 프로그래밍 분야 이다.

 ( d ) 상태 ( state ) 란 무엇인가? 순수 함수형 프로그래밍에서는 왜 상태가 발생하지 않는가?
- 상태 : 상황을 알려 주기 위해서 변수이다.
- 받은 인자 외에 다른 외부의 상태에 영향을 끼치않고 리턴값 외에는 외부와 소통이 없다. 따라서 상태가 발생하지 않는다.
